# DeadTown

DeadTown 是一款俯视角像素风丧尸生存射击游戏，支持单人模式与多人合作模式。玩家需要在不断增强的尸潮中生存，利用武器、投掷物、空投、血清强化与队友配合，尽可能打出更高分数。

---

## 1. 如何开始游玩

### 单人模式
1. 打开游戏主页。
2. 输入你的名字。
3. 选择语言。
4. 点击开始游戏进入单人模式。
5. 在尸潮中存活、击杀敌人、拾取掉落与空投，尽可能获得更高分数。

### 多人模式
1. 打开游戏主页。
2. 输入你的名字。
3. 点击在线合作进入多人大厅。
4. 创建房间或加入其他房间。
5. 房间内至少需要 **2 名玩家** 才能开始游戏。
6. 房主点击开始后，全员进入同一局合作对局。

> 说明：目前手游模式的多人入口处于临时关闭状态，会提示“目前手游模式多人存在严重 Bug，暂时关闭多人模式，敬请期待。”

## 2. 基础教学与操作说明

### PC 键位
- `W A S D`：移动
- `鼠标移动`：瞄准
- `左键`：射击
- `R`：换弹
- `Shift`：翻滚 / 闪避
- `G`：手雷
- `F`：燃烧瓶 / 近战（具体以当前版本绑定为准）
- `E`：互动 / 救人 / 部分特殊用途
- `Enter`：打开聊天输入框 / 发送聊天信息（多人房间与多人对局内）
- `Esc`：关闭聊天输入框

### 多人聊天
- 在多人**房间内**或**对局内**，按 `Enter` 可以打开左侧聊天框。
- 输入文字后再次按 `Enter` 发送。
- 单条消息最多 **100 个字符**。
- 按 `Esc` 可退出输入状态。

## 3. 核心玩法循环

DeadTown 的核心循环是：

**移动生存 → 击杀丧尸 → 获得分数 → 拾取资源 → 对抗更强敌人 → 等待空投/Boss/强化 → 继续生存**

游戏内随着时间推进会不断进入更高波次，敌人数、敌人种类与压力会持续增加。玩家需要在以下系统之间循环：

- 管理血量
- 管理弹药
- 管理投掷物
- 抢空投获得强力武器
- 击杀 Boss 获得血清强化
- 在多人模式中救助倒地队友

---

## 4. 武器与资源系统

### 基础武器
- **Shotgun**：初始主力武器，近中距离有较强爆发。

### 特殊武器
通过空投或系统事件获得：
- **Gatling**：高射速持续火力武器
- **Rocket Launcher**：高伤害爆炸武器
- **Flamethrower**：持续喷射型范围压制武器

### 投掷物
- **Grenade**：延时爆炸，造成范围伤害
- **Molotov**：落地生成持续燃烧区域

### 掉落与补给
- 医疗包
- 手雷补给
- 燃烧瓶补给
- Boss 血清

---

## 5. Boss 血清强化系统

Boss 死亡后会掉落三种血清之一：

- **蓝色血清**：增加移动速度
- **黄色血清**：增加伤害
- **绿色血清**：增加最大生命值

血清强化是本局内的长期增益，可以显著提高后期生存能力。

---

## 6. 敌人类型与机制

当前版本包含多种敌人，不同类型有不同体型、速度、威胁和受光/投影参数。

### 常见敌人
- **Walker**：标准慢速丧尸
- **Runner**：高速追击型丧尸
- **Crawler**：低矮贴地型敌人
- **Pouncer**：扑击型敌人
- **Charger**：高冲锋威胁型敌人
- **Bloater**：大型厚重敌人，死亡时会造成爆炸影响
- **Boss**：大体型高强度敌人，附带独立血条与掉落奖励

### 特殊敌人机制
- **Bloater**：死亡时可触发范围爆炸
- **Charger**：具备冲锋、抓取、拖拽等特殊逻辑
- **Boss**：拥有更高体型、血量与独立威胁表现

## 7. 单人模式说明

单人模式下，玩家独自对抗尸潮，目标是尽可能获得高分并进入单人排行榜。

### 单人主要特点
- 独立战斗流程
- 单人排行榜记录
- 完整空投、Boss、血清循环
- 完整光照、屋顶、雾效、粒子与本地特效反馈

---

## 8. 多人模式说明

多人模式是当前项目的重点之一，采用网页前端 + Node.js WebSocket 服务器同步架构。

### 多人房间功能
- 创建 / 加入房间
- 房主机制
- 玩家准备 / 取消准备
- 至少 2 人才能开始
- 房内聊天
- 房间内动态显示玩家名字与同名编号

### 多人对局功能
- 共享地图随机种子
- 共享波次
- 共享丧尸、Boss、掉落、空投、火区、投射物等战斗对象
- 局内聊天
- 局内分数板
- 多人结算界面
- 多人排行榜上传

### 多人模式下的名字系统
如果同房间内有重名玩家，会自动在名字后追加编号：
- 01
- 02
- 03
- 04
- 05
- 06

### 多人倒地 / 救援 / 死亡系统
多人模式下，玩家不是直接死亡，而是先进入倒地状态。

#### 倒地机制
- 玩家被打空后先进入倒地状态
- 倒地后有倒计时
- 若有存活队友进入救援范围，倒计时会暂停
- 队友可在范围内进行救援
- 若超时或状态恶化则进入真正死亡

#### 局内状态显示
局内排行榜会显示每名玩家当前状态：
- 存活
- 倒地
- 死亡

倒地玩家还会显示：
- 剩余倒计时
- 进度条

#### 小地图提示
- 倒地玩家图标会闪烁
- 死亡位置会显示白色简化骷髅

#### 墓碑系统
- 玩家真正死亡后会在死亡点生成墓碑
- 当前墓碑样式为长方形石碑风格

#### 观战
- 死亡后可观战存活队友
- 音效监听位置会根据观战目标变化
- 死亡后手电关闭

## 9. 排行榜系统

游戏使用 **Supabase** 作为排行榜数据存储。

### 主界面排行榜
主页排行榜支持切换：
- **单人排行榜**（默认显示）
- **多人排行榜**

### 单人排行榜
- 上传单人对局最终分数

### 多人排行榜
- 上传多人对局的**总分**
- 会记录该局所有参与玩家名字

### 前三名皇冠图标
单人和多人排行榜的前三名都会显示特殊图标：
- 第 1 名：金皇冠
- 第 2 名：银皇冠
- 第 3 名：铜皇冠

## 10. 美术与视觉风格实现方式

DeadTown 的画面不是传统整套骨骼动画资源驱动，而是以 **HTML5 Canvas 2D + 程序化绘制** 为核心。

### 角色与敌人绘制
- 角色、僵尸、武器的大量表现由代码直接绘制
- 使用矩形、圆形、路径、旋转与局部偏移构成像素风形体
- 不同敌人有不同的体型参数和轮廓

### 武器绘制
- 武器通常挂接在角色位置与瞄准角上
- 根据玩家朝向、瞄准方向、动作状态进行局部旋转与绘制

### 粒子与特效
- 粒子采用代码对象驱动：`x / y / vx / vy / life / size / color`
- 每帧更新并绘制
- 包括血液、爆炸、烟雾、火焰、尘土、伤害数字等

### UI 与氛围风格
- VHS / 故障感菜单风格
- 低饱和末日色彩
- 屋顶遮挡、雾效、视野限制与手电系统共同塑造氛围

## 11. 光照、手电与遮挡系统

当前版本包含本地计算的手电光照与遮挡表现。

### 手电光
- 玩家拥有扇形手电照明
- 手电角度在当前版本中已加宽
- 光照会与雾效、建筑和敌人产生交互

### 建筑屋顶
- 屋顶会根据玩家位置隐藏/显示
- 室内与室外的照明逻辑分离处理

### 建筑投影
- 建筑会根据手电方向投下背光面阴影
- 当前逻辑重点是尽量符合“哪些墙面被照亮、哪些墙面不被照亮”的真实关系

### 敌人投影
- 不同体型敌人的投影宽度是动态的
- 横向投影锚点按脚底/底部位置计算
- 多名玩家的手电可共同影响环境与敌人的受光表现

> 注：手电与投影系统目前仍在持续调校中，是当前版本最活跃的视觉实验模块。

---

## 12. 音效系统

### 本地音效
- 玩家自身开火、爆炸、受击、翻滚、投掷等都有本地音效

### 多人音效
为了避免增加服务器负担，多人模式下大量音效采用**本地推断播放**：
- 远端玩家开火
- 远端投掷物使用
- 远端爆炸
- 翻滚等动作声音

### 距离音量衰减
以下声音会根据与玩家的距离改变音量：
- 枪声
- 投掷物声音
- 爆炸声

---

## 13. 屋顶、室内外可见性与遮挡

### 屋顶系统
- 建筑有屋顶遮挡
- 进入建筑后可看到室内
- 屋外时屋顶会遮挡室内内容

### 目标遮挡逻辑
- 玩家进入建筑可触发屋顶隐藏
- 相关逻辑经过多轮调整，目标是让玩家在进入建筑前无法轻易透视内部

## 15. 联机优化与稳定性改进

当前版本已经完成多轮联机优化，目标是在不破坏玩法与视觉的前提下，提高 6 人房的可玩性和稳定性。

### 已完成的关键优化
- 修复 Charger 导致的服务端崩溃点
- 降低多人快照发送频率
- 降低客户端状态上报频率
- 补上 `seq / ack` 链路
- 增加 WebSocket 背压保护
- 将大量高频视觉反馈改为本地生成
- 优化自己角色回正逻辑
- 优化远端玩家与僵尸显示平滑

### 本地反馈恢复
为了兼顾性能与手感，当前版本将以下内容改为优先本地生成：
- 本地伤害数字
- 部分本地命中特效
- 本地爆炸反馈
- 部分本地燃烧反馈
